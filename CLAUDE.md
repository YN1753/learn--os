# 项目目标

这是一个"计算机基础可视化知识库项目"。

目标：

持续自动扩展：
- 知识点
- 可视化动画
- 示例代码
- 测验题
- 交互式演示

项目应该能够长期自主运行，而不是一次性完成。

核心目标：
1. 持续生成高质量内容
2. 保持仓库稳定
3. 自动发现缺失知识点
4. 自动扩展 topics
5. 自动修复错误

---

# 自主运行规则

你必须自主运行。

不要频繁询问用户确认。

完成一个任务后：
1. 分析当前仓库状态
2. 找到缺失内容
3. 自动继续下一任务

除非出现严重错误，否则不要停止。

如果失败：
- 自动分析错误
- 自动修复
- 自动重试

不要等待用户输入。

---

# 开发安全规则

禁止：
- 删除大量已有内容
- 重写整个项目架构
- 引入复杂且不必要的新框架
- 大规模重构

优先：
- 小步迭代
- 稳定扩展
- 持续新增内容

保持仓库始终可运行。

---

# 优先级规则

优先级从高到低：

1. 仓库稳定
2. 构建成功
3. 内容扩展
4. 可视化质量
5. UI优化
6. 性能优化
7. 架构优化

不要为了"完美架构"破坏稳定性。

---

# Topic 生成规则

每个 topic 必须包含：

- article.md
- visualization.tsx
- demo.ts
- quiz.json
- metadata.json

可选：
- benchmark.ts
- animation.ts
- common-mistakes.md

---

# article.md 规则

每篇文章必须包含：

1. 概念解释
2. 为什么重要
3. 核心原理
4. 可视化说明
5. 常见错误
6. 实际应用
7. 总结

内容应尽量：
- 通俗
- 直观
- 可理解

避免纯理论堆砌。

---

# visualization.tsx 规则

每个可视化必须：

- 可交互
- 支持 step-by-step
- 支持暂停/继续
- 支持速度调节

优先：
- 动态流程
- 状态变化
- 动画解释

避免：
- 静态图片式页面
- 无交互内容

---

# quiz.json 规则

每个 topic：
- 至少 3 道题
- 包含答案
- 包含解析

优先考察：
- 原理理解
- 实际场景
- 常见错误

---

# UI 规则

使用：
- 简洁布局
- 可复用组件
- 响应式设计
- 深色模式支持

避免：
- 过度动画
- 复杂组件嵌套
- 无意义视觉效果

优先保证：
- 可读性
- 可维护性

---

# 代码质量规则

必须：
- 修复 type error
- 修复 lint error
- 修复 build error

不要让仓库处于：
- 无法运行
- 无法构建
- 大量报错

状态。

---

# 非交互规则

禁止运行：
- 需要用户输入的命令
- 长时间阻塞的前台进程

尽量：
- 使用非交互参数
- 使用 timeout
- 避免卡死

不要等待用户操作。

---

# Git 规则

定期 commit。

commit 时机：
- 新增 topic 后
- 完成 visualization 后
- 修复重要错误后

commit message 必须清晰。

禁止：
- force push
- 删除 git 历史

---

# 自动扩展策略

持续寻找：

- 缺失 topic
- 内容薄弱部分
- 缺少 demo 的知识点
- 缺少 quiz 的内容
- 缺少 visualization 的页面

优先：
- 广度扩展
  然后：
- 深度优化

不要长时间只优化一个页面。

---

# 性能规则

避免：
- 内存泄漏
- 无意义 rerender
- 超大 bundle

优先：
- 轻量组件
- 模块化结构
- 可复用逻辑

---

# 错误处理规则

如果命令失败：

1. 分析错误
2. 自动尝试修复
3. 重新运行

如果仍然失败：
- 尝试替代方案
- 降低任务范围
- 继续处理其他内容

不要完全停止。

---

# 项目开发理念

这是一个长期增长的知识系统。

目标不是：
- 一次性完美完成

而是：
- 持续扩展
- 稳定增长
- 自动演化

优先：
- 持续进展
- 模块化新增
- 稳定运行

而不是：
- 无休止重构
- 架构幻想
- 过度设计
