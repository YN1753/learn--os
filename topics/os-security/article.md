# 操作系统安全

## 概念解释

操作系统安全是保护计算机系统免受未经授权的访问、使用、泄露、破坏或修改的措施和机制。

你可以把操作系统安全想象成**房子的安保系统**：
- 门锁（认证）：确认你是房主
- 钥匙分配（授权）：谁能进哪些房间
- 监控（审计）：记录谁做了什么
- 防盗门（防护）：阻止入侵者

## 为什么重要

没有安全保护：
- 恶意软件可以随意破坏系统
- 他人可以窃取你的数据
- 系统可能被远程控制
- 隐私完全暴露

有了安全保护：
- 只有授权用户能访问系统
- 用户只能访问自己的数据
- 恶意行为被阻止或记录
- 系统稳定运行

## 安全威胁类型

### 1. 恶意软件（Malware）

**病毒**：附着在正常程序上，感染其他程序。
**蠕虫**：独立传播，通过网络扩散。
**木马**：伪装成正常软件，实际有恶意功能。
**勒索软件**：加密用户文件，索要赎金。
**间谍软件**：偷偷收集用户信息。

### 2. 网络攻击

**拒绝服务攻击（DoS）**：让系统无法正常服务。
**中间人攻击**：拦截通信双方的数据。
**端口扫描**：探测系统开放的服务。

### 3. 社会工程学

**钓鱼攻击**：伪装成可信来源骗取信息。
**假冒身份**：冒充他人获取权限。

## 认证机制

### 什么是认证？

认证是验证用户身份的过程。"你是谁？"

### 认证因素

**你知道的**：密码、PIN码
**你拥有的**：手机、智能卡、U盾
**你本身的**：指纹、人脸、虹膜

### 多因素认证

结合两种以上因素，如密码+手机验证码。

### 密码安全

**存储**：
- 不存储明文密码
- 使用哈希算法（如bcrypt、SHA-256）
- 加盐（salt）防止彩虹表攻击

**策略**：
- 最小长度要求
- 复杂度要求
- 定期更换
- 禁止重复使用

## 访问控制

### 什么是访问控制？

访问控制决定谁能访问什么资源。"你能做什么？"

### 访问控制模型

**1. 自主访问控制（DAC）**
- 资源所有者决定谁能访问
- 灵活但安全性较低
- 如Linux文件权限

**2. 强制访问控制（MAC）**
- 系统统一管理访问权限
- 安全性高但不灵活
- 如军事安全级别

**3. 基于角色的访问控制（RBAC）**
- 按角色分配权限
- 用户属于某个角色
- 如管理员、普通用户、访客

**4. 基于属性的访问控制（ABAC）**
- 根据属性决定访问权限
- 如时间、地点、设备状态
- 最灵活

### Linux权限模型

**用户分类**：
- 所有者（u）
- 组（g）
- 其他（o）

**权限类型**：
- 读（r）：查看内容
- 写（w）：修改内容
- 执行（x）：运行程序

**示例**：
```
-rwxr-xr-- 1 user group 4096 file.txt
│ │  │  │
│ │  │  └── 其他用户：只读
│ │  └───── 组：读和执行
│ └──────── 所有者：读写执行
└────────── 文件类型
```

## 加密技术

### 对称加密

加密和解密使用同一个密钥。

**常见算法**：AES、DES、3DES

**特点**：速度快，但密钥分发困难。

### 非对称加密

使用公钥加密，私钥解密。

**常见算法**：RSA、ECC

**特点**：安全但速度慢。

### 实际应用

**HTTPS**：
- 用非对称加密交换密钥
- 用对称加密传输数据
- 兼顾安全和性能

**数字签名**：
- 用私钥签名，公钥验证
- 确保数据完整性和不可否认性

## 安全机制

### 1. 最小权限原则

每个程序和用户只拥有完成任务所需的最小权限。

### 2. 隔离

- **进程隔离**：进程之间内存独立
- **用户隔离**：不同用户有独立空间
- **网络隔离**：不同网络区域分开

### 3. 审计

记录系统活动，用于事后分析。

**日志类型**：
- 登录日志
- 文件访问日志
- 系统事件日志

### 4. 防火墙

控制进出网络的数据包。

**类型**：
- 包过滤防火墙
- 状态检测防火墙
- 应用层防火墙

### 5. 入侵检测系统（IDS）

监控系统活动，检测可疑行为。

**类型**：
- 基于签名：匹配已知攻击模式
- 基于异常：检测偏离正常的行为

## 常见安全实践

### 系统加固

1. 及时安装安全补丁
2. 关闭不需要的服务
3. 使用强密码策略
4. 配置防火墙
5. 启用审计日志

### 安全开发

1. 输入验证：防止注入攻击
2. 最小权限：程序不请求多余权限
3. 安全存储：敏感数据加密
4. 错误处理：不泄露系统信息

## 常见错误

**错误1：安装了杀毒软件就安全了**

杀毒软件只是安全体系的一部分，还需要防火墙、更新补丁、安全意识等。

**错误2：复杂密码就安全了**

密码安全还包括：不在多处使用相同密码、定期更换、不泄露给他人。

**错误3：内网不需要安全防护**

内部威胁同样危险，很多安全事件来自内部人员。

## 实际应用

**Windows**：用户账户控制（UAC）、Windows Defender、BitLocker加密。

**Linux**：SELinux、AppArmor、iptables防火墙。

**手机**：指纹/人脸解锁、应用沙箱、权限管理。

**企业**：零信任架构、多因素认证、安全审计。

## 总结

- 操作系统安全保护系统免受各种威胁
- 认证确认用户身份，授权控制用户权限
- 访问控制模型：DAC、MAC、RBAC、ABAC
- 加密技术保护数据的机密性和完整性
- 安全机制包括最小权限、隔离、审计、防火墙等
- 安全是一个持续的过程，需要多层防护
