# 进程创建与生命周期

## 概念解释

**进程**是正在执行的程序。操作系统通过系统调用创建、管理和终止进程。理解进程的创建过程和生命周期是理解操作系统的基础。

就像一个公司：创建新部门（fork），分配具体工作（exec），部门完成工作后解散（exit）。

## 为什么重要

- **多任务的基础**：操作系统通过进程实现并发执行
- **资源隔离**：每个进程有独立的地址空间
- **理解僵尸进程**：不当的进程管理会导致资源泄漏

## 核心系统调用

### 1. fork() - 创建子进程

```c
pid_t pid = fork();
if (pid == 0) {
    // 子进程代码
} else {
    // 父进程代码，pid 是子进程的 PID
}
```

**特点**：
- 子进程是父进程的副本（写时复制）
- 子进程获得父进程的文件描述符
- fork() 在父进程中返回子进程 PID，在子进程中返回 0

### 2. exec() - 替换进程映像

```c
execl("/bin/ls", "ls", "-l", NULL);
```

**特点**：
- 用新程序替换当前进程的代码、数据和堆栈
- 进程 ID 不变
- 不会创建新进程

### 3. exit() - 终止进程

```c
exit(0);  // 正常退出
exit(1);  // 异常退出
```

**特点**：
- 关闭所有打开的文件描述符
- 向父进程发送 SIGCHLD 信号
- 进程进入僵尸状态等待父进程回收

### 4. wait() - 等待子进程

```c
int status;
pid_t pid = wait(&status);
```

**特点**：
- 阻塞等待子进程结束
- 回收子进程资源
- 获取子进程退出状态

## 进程状态

```
创建 → 就绪 → 运行 → 终止
         ↑      ↓
         └ 阻塞 ←┘
```

- **创建（New）**：进程正在被创建
- **就绪（Ready）**：等待 CPU
- **运行（Running）**：正在 CPU 上执行
- **阻塞（Blocked）**：等待某个事件（如 IO）
- **终止（Terminated）**：进程执行完毕

## 僵尸进程

当子进程退出但父进程没有调用 wait() 回收时，子进程成为僵尸进程：
- 进程已终止，但 PCB 仍在进程表中
- 占用系统资源
- 最终可能耗尽进程表

## 可视化说明

在可视化中，你可以：
- 观察 fork() 创建子进程的过程
- 理解进程状态的转换
- 模拟僵尸进程的产生和处理

## 常见错误

1. **忘记 wait()**：导致僵尸进程
2. **fork 后不检查返回值**：父子进程执行相同代码
3. **exec 后的代码**：exec 成功后不会返回，后面的代码不会执行

## 实际应用

- **Shell**：fork + exec 执行命令
- **Web 服务器**：fork 子进程处理请求
- **守护进程**：fork 两次创建后台进程

## 总结

进程创建通过 fork + exec 模式实现。理解进程生命周期和状态转换，有助于编写正确的多进程程序和避免僵尸进程等问题。
