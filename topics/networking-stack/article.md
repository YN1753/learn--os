# 网络协议栈

## 概念解释

**网络协议栈**是计算机网络通信的分层模型。每一层负责不同的功能，共同实现网络通信。

就像邮政系统：写信（应用层）→ 装信封（传输层）→ 写地址（网络层）→ 投递（链路层）。

## 为什么重要

- **理解网络通信**：数据如何从A到B
- **故障排查**：定位网络问题
- **性能优化**：优化网络配置

## TCP/IP 四层模型

### 1. 应用层

用户直接使用的协议。

- **HTTP/HTTPS**：网页浏览
- **FTP**：文件传输
- **SMTP**：邮件发送
- **DNS**：域名解析

### 2. 传输层

端到端通信。

- **TCP**：可靠传输，面向连接
- **UDP**：不可靠传输，无连接

### 3. 网络层

路由和寻址。

- **IP**：数据包寻址
- **ICMP**：错误报告
- **ARP**：地址解析

### 4. 链路层

物理传输。

- **以太网**：局域网
- **WiFi**：无线网络

## 数据封装

```
应用层数据
  ↓ 加TCP头
TCP段
  ↓ 加IP头
IP包
  ↓ 加以太网头
以太网帧
```

## TCP 三次握手

```
客户端 → SYN → 服务器
客户端 ← SYN+ACK ← 服务器
客户端 → ACK → 服务器
```

## 可视化说明

在可视化中，你可以：
- 观察数据封装过程
- 理解TCP三次握手
- 了解各层协议的作用

## 常见错误

1. **混淆TCP和UDP**：TCP可靠但慢，UDP快但不可靠
2. **忽略MTU**：数据包大小限制
3. **端口冲突**：同一端口只能被一个程序监听

## 实际应用

- **Web开发**：HTTP协议
- **游戏**：UDP协议
- **视频流**：UDP或TCP

## 总结

网络协议栈通过分层实现网络通信。理解各层的作用和协议，有助于开发网络应用和排查网络问题。
