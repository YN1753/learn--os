# 调度算法详解

## 概念解释

**调度算法**决定哪个进程获得 CPU 使用权。不同的算法有不同的优化目标：吞吐量、响应时间、公平性等。

就像银行叫号系统：可以按先来后到（FCFS），也可以按业务复杂度（SJF），或者VIP优先（优先级）。

## 为什么重要

- **系统性能的核心**：调度算法直接影响系统效率
- **用户体验**：响应时间影响交互体验
- **资源利用**：好的调度提高 CPU 利用率

## 调度算法分类

### 1. FCFS（先来先服务）

按到达顺序服务。

```
进程  到达  执行时间
P1    0     24
P2    1     3
P3    2     3

执行顺序: P1 → P2 → P3
平均等待时间: (0 + 23 + 24) / 3 = 15.67
```

**优点**：简单，公平
**缺点**：平均等待时间长， convoy effect

### 2. SJF（最短作业优先）

选择执行时间最短的进程。

```
执行顺序: P2 → P3 → P1
平均等待时间: (6 + 0 + 0) / 3 = 2
```

**优点**：平均等待时间最短
**缺点**：可能导致长作业饥饿

### 3. SRTF（最短剩余时间优先）

SJF 的抢占版本：新进程到达时，如果剩余时间更短则抢占。

### 4. RR（时间片轮转）

每个进程分配固定时间片，时间片用完则切换到下一个。

```
时间片 = 4

执行顺序: P1(4) → P2(3) → P3(3) → P1(4) → ...
```

**优点**：响应时间短，公平
**缺点**：时间片太小导致频繁切换

### 5. 优先级调度

每个进程有优先级，高优先级先执行。

```
进程  优先级
P1    3 (低)
P2    1 (高)
P3    2 (中)

执行顺序: P2 → P3 → P1
```

**优点**：灵活，可以表达重要性
**缺点**：低优先级可能饥饿

### 6. 多级反馈队列

结合多种算法：多个队列，不同队列有不同时间片。

```
高优先级队列: 时间片小 (响应型)
中优先级队列: 时间片中
低优先级队列: 时间片大 (吞吐型)
```

**优点**：兼顾响应时间和吞吐量
**缺点**：实现复杂

## 性能指标

| 指标 | 说明 | 优化目标 |
|------|------|----------|
| 吞吐量 | 单位时间完成的进程数 | 越大越好 |
| 响应时间 | 从请求到首次响应 | 越小越好 |
| 等待时间 | 在就绪队列中等待的总时间 | 越小越好 |
| 周转时间 | 从提交到完成的总时间 | 越小越好 |
| CPU 利用率 | CPU 忙碌时间占比 | 越高越好 |

## 可视化说明

在可视化中，你可以：
- 比较不同算法的执行过程
- 观察甘特图和性能指标
- 理解抢占和非抢占的区别

## 常见错误

1. **忽略到达时间**：SJF 不是简单排序，要考虑到达时间
2. **时间片选择**：太小导致切换开销，太大退化为 FCFS
3. **饥饿问题**：优先级调度需要老化机制

## 实际应用

- **Linux CFS**：基于虚拟运行时间的公平调度
- **Windows**：多级反馈队列，32 个优先级
- **实时系统**：EDF（最早截止时间优先）

## 总结

不同调度算法有不同的适用场景。FCFS 简单但效率低，SJF 理论最优但不公平，RR 兼顾响应和公平，多级反馈队列是实际系统的常用选择。
