# 负载均衡

## 概念解释

负载均衡是将网络流量分配到多个服务器的技术，目的是提高系统可用性和响应速度。负载均衡器位于客户端和服务器之间，根据算法选择后端服务器。

## 为什么重要

- **高可用**：单点故障不影响服务
- **扩展性**：通过增加服务器提高处理能力
- **性能**：分散请求，减少单机压力
- **灵活性**：支持动态增减服务器

## 核心原理

### 负载均衡算法

| 算法 | 说明 |
|------|------|
| 轮询(Round Robin) | 依次分配请求 |
| 加权轮询 | 按权重分配 |
| 最少连接 | 选择连接数最少的服务器 |
| IP哈希 | 同一IP分配到同一服务器 |
| 随机 | 随机选择服务器 |

### 负载均衡层次

| 层次 | 协议 | 示例 |
|------|------|------|
| L4(传输层) | TCP/UDP | LVS |
| L7(应用层) | HTTP | Nginx, HAProxy |

### 健康检查

- **主动检查**：定期发送探测请求
- **被动检查**：根据响应判断健康状态
- **自动剔除**：故障服务器自动移除

## 可视化说明

可视化组件展示了负载均衡算法和请求分发过程。

## 常见错误

1. **会话丢失**：轮询算法导致Session问题
2. **单点故障**：负载均衡器本身成为瓶颈
3. **健康检查不当**：未及时剔除故障服务器

## 实际应用

- **Web应用**：Nginx反向代理
- **数据库**：读写分离
- **微服务**：服务间负载均衡
- **CDN**：内容分发

## 总结

负载均衡是构建高可用系统的关键技术，选择合适的算法和实现方式对系统性能至关重要。
