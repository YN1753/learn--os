# 分布式算法

## 概念解释

分布式算法是在分布式系统中多个节点之间达成一致的算法。共识算法（如Raft、Paxos）确保多个节点对某个值达成一致，即使部分节点故障。

## 为什么重要

- **数据一致性**：确保分布式系统数据一致
- **高可用**：部分节点故障系统仍可运行
- **分布式存储**：如etcd、ZooKeeper等依赖共识算法
- **区块链**：区块链共识机制的基础

## 核心原理

### Raft算法角色

| 角色 | 职责 |
|------|------|
| Leader | 处理所有写请求，复制日志 |
| Follower | 响应Leader请求 |
| Candidate | 发起选举 |

### Raft选举过程

1. Follower超时未收到心跳
2. 转为Candidate，发起投票
3. 获得多数票成为Leader
4. 定期发送心跳维持领导

### 日志复制

```
Client → Leader → 写入日志 → 复制到Follower → 多数确认 → 提交
```

### CAP定理

| 属性 | 说明 |
|------|------|
| 一致性(C) | 所有节点看到相同数据 |
| 可用性(A) | 每个请求都能得到响应 |
| 分区容忍(P) | 网络分区时系统仍能运行 |

分布式系统最多只能同时满足两个属性。

## 可视化说明

可视化组件展示了Raft选举过程和日志复制。

## 常见错误

1. **忽视网络分区**：不处理网络分区导致脑裂
2. **超时设置不当**：选举超时太短导致频繁选举
3. **日志不一致**：未正确处理日志冲突

## 实际应用

- **etcd**：Kubernetes使用的分布式键值存储
- **ZooKeeper**：分布式协调服务
- **Consul**：服务发现和配置
- **CockroachDB**：分布式数据库

## 总结

分布式算法是分布式系统的基石，理解Raft等共识算法对于构建可靠的分布式系统至关重要。
