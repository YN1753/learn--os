# 文件权限管理

## 概念解释

**文件权限**控制谁可以访问文件以及如何访问。Unix/Linux使用简单而强大的权限模型。

就像房间的钥匙：房主有所有钥匙，家人有一些，外人没有。

## 为什么重要

- **安全性**：防止未授权访问
- **隐私**：保护个人文件
- **系统稳定**：防止误操作

## Unix 权限模型

### 三类用户

- **所有者（Owner）**：文件创建者
- **组（Group）**：文件所属用户组
- **其他（Others）**：所有人

### 三种权限

- **读（r = 4）**：查看文件内容
- **写（w = 2）**：修改文件内容
- **执行（x = 1）**：运行文件

### 权限表示

```
-rwxr-xr--
│ │  │  │
│ │  │  └── 其他用户：只读
│ │  └───── 组用户：读+执行
│ └──────── 所有者：读+写+执行
└────────── 文件类型（- 普通文件，d 目录）
```

数字表示：754 = rwxr-xr--

## 权限管理命令

### chmod - 修改权限

```bash
chmod 755 file      # 数字方式
chmod u+x file      # 符号方式：给所有者添加执行权限
chmod g-w file      # 移除组的写权限
chmod o=r file      # 设置其他为只读
```

### chown - 修改所有者

```bash
chown user file
chown user:group file
```

### chgrp - 修改组

```bash
chgrp group file
```

## 特殊权限

### 1. SUID (4000)

执行时以文件所有者身份运行。

```bash
chmod u+s program
```

### 2. SGID (2000)

执行时以文件组身份运行。

### 3. Sticky Bit (1000)

只有所有者能删除文件（如 /tmp 目录）。

```bash
chmod +t directory
```

## ACL（访问控制列表）

更细粒度的权限控制：

```bash
setfacl -m u:alice:rw file    # 给alice读写权限
getfacl file                  # 查看ACL
```

## 可视化说明

在可视化中，你可以：
- 理解权限位的含义
- 练习权限管理命令
- 理解特殊权限的作用

## 常见错误

1. **权限过大**：给777权限很危险
2. **忽略目录权限**：目录需要执行权限才能进入
3. **SUID风险**：SUID程序可能被利用

## 实际应用

- **Web服务器**：www-data用户权限
- **共享目录**：组权限管理
- **系统安全**：最小权限原则

## 总结

文件权限是系统安全的基础。理解rwx权限、chmod命令和特殊权限，有助于保护系统安全。
