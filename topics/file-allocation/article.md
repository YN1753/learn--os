# 文件分配策略

## 概念解释

**文件分配策略**决定文件数据如何存储在磁盘块中。不同的策略在访问速度、空间利用和支持随机访问方面有不同的权衡。

就像图书馆的书架管理：可以按顺序放（连续分配），可以链式放（链接分配），或者用目录卡（索引分配）。

## 为什么重要

- **访问速度**：直接影响文件读写性能
- **空间利用**：好的策略减少浪费
- **文件大小**：影响支持的最大文件大小

## 连续分配

文件占据连续的磁盘块。

```
文件A: 块0-3
文件B: 块5-8
文件C: 块10-12
```

**优点**：
- 顺序和随机访问都快
- 实现简单

**缺点**：
- 外部碎片
- 文件大小难以扩展

## 链接分配

每个块指向下一个块，形成链表。

```
文件A: 块2 → 块5 → 块1 → 块8
```

**优点**：
- 无外部碎片
- 文件大小可动态增长

**缺点**：
- 只能顺序访问
- 指针占用空间
- 可靠性问题（一个块坏，后面全丢）

### FAT（文件分配表）

将链接信息集中存储在表中：

```
块号:  0  1  2  3  4  5  6  7  8
下一:  5  8  -1  -  -1 1  -  -  2
```

**优点**：支持随机访问（通过FAT表）
**缺点**：FAT表需要内存

## 索引分配

为每个文件创建索引块，存储所有数据块指针。

```
索引块: [0, 2, 5, 8]
数据块: 块0, 块2, 块5, 块8
```

**优点**：
- 支持随机访问
- 无外部碎片

**缺点**：
- 索引块开销
- 大文件需要多级索引

### 多级索引

索引块指向索引块，支持大文件：

```
一级索引 → 二级索引 → 数据块
```

## 可视化说明

在可视化中，你可以：
- 观察不同分配方式的磁盘布局
- 理解文件访问过程
- 比较各种策略的优缺点

## 常见错误

1. **连续分配的碎片**：频繁创建删除文件导致碎片
2. **链接分配的可靠性**：一个块损坏导致整个文件丢失
3. **索引分配的开销**：小文件也需要索引块

## 实际应用

- **FAT32**：使用FAT表，兼容性好
- **NTFS**：使用B+树索引
- **ext4**：使用多级索引
- **SSD**：使用FTL（闪存转换层）

## 总结

连续分配快但有碎片，链接分配灵活但只支持顺序访问，索引分配两者兼顾但有开销。现代文件系统通常使用索引分配的变种。
