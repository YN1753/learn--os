# 系统调用

## 概念解释

系统调用是程序请求操作系统服务的接口。

想象酒店：
- 你（用户程序）不能直接进厨房
- 你通过服务员（系统调用）点菜
- 厨房（内核）做好后服务员给你端来

## 为什么重要

系统调用是程序和操作系统之间的桥梁：
- 程序不能直接访问硬件
- 程序不能直接操作其他程序的内存
- 程序需要通过系统调用请求服务

## 用户态与内核态

### 两种运行模式

**用户态**：
- 运行应用程序
- 权限受限
- 不能直接访问硬件
- 不能执行特权指令

**内核态**：
- 运行操作系统内核
- 拥有最高权限
- 可以访问所有资源
- 可以执行所有指令

### 为什么要区分？

- **安全性**：防止程序破坏系统
- **稳定性**：防止程序影响其他程序
- **隔离性**：每个程序有自己的地址空间

### 状态切换

用户态 → 内核态：通过中断或系统调用
内核态 → 用户态：系统调用完成后自动切换

## 系统调用的分类

### 进程管理

- `fork()`：创建子进程
- `exec()`：执行程序
- `exit()`：终止进程
- `wait()`：等待子进程

### 文件操作

- `open()`：打开文件
- `read()`：读取文件
- `write()`：写入文件
- `close()`：关闭文件

### 内存管理

- `mmap()`：内存映射
- `brk()`：调整堆大小
- `malloc()`：分配内存（库函数，最终调用系统调用）

### 设备管理

- `ioctl()`：设备控制
- `read()`：从设备读取
- `write()`：向设备写入

### 网络通信

- `socket()`：创建套接字
- `bind()`：绑定地址
- `listen()`：监听连接
- `accept()`：接受连接
- `connect()`：建立连接

## 系统调用的执行过程

1. 用户程序调用库函数
2. 库函数将系统调用号放入寄存器
3. 触发软件中断（陷阱）
4. CPU切换到内核态
5. 内核查找系统调用表
6. 执行对应的系统调用处理程序
7. 将结果放入寄存器
8. 返回用户态
9. 库函数返回结果给用户程序

## 系统调用 vs 库函数

| 特性 | 系统调用 | 库函数 |
|------|----------|--------|
| 运行态 | 内核态 | 用户态 |
| 开销 | 大（状态切换） | 小 |
| 功能 | 底层操作 | 封装功能 |
| 示例 | `read()` | `fread()` |

**关系**：库函数通常封装系统调用，提供更方便的接口。

## 常见系统调用详解

### fork()

创建一个新进程，新进程是父进程的副本。

```c
pid_t pid = fork();
if (pid == 0) {
    // 子进程
} else {
    // 父进程
}
```

### exec()

用新程序替换当前进程的内存空间。

```c
execl("/bin/ls", "ls", "-l", NULL);
```

### open() / read() / write()

文件操作的三部曲：

```c
int fd = open("file.txt", O_RDONLY);
char buf[1024];
int n = read(fd, buf, sizeof(buf));
close(fd);
```

### pipe()

创建管道，用于进程间通信。

```c
int pipefd[2];
pipe(pipefd);
// pipefd[0] 读端
// pipefd[1] 写端
```

## 系统调用的性能

### 开销来源

- 用户态和内核态切换
- 保存和恢复寄存器
- 内存空间切换
- 安全检查

### 优化方法

- **批量处理**：减少系统调用次数
- **缓存**：避免重复系统调用
- **vDSO**：将部分系统调用放在用户态

## 常见错误

**错误1：系统调用和普通函数一样**

系统调用有状态切换开销，应该尽量减少调用次数。

**错误2：所有操作都需要系统调用**

很多操作可以用库函数完成，不需要系统调用。

**错误3：系统调用总是成功的**

系统调用可能失败，必须检查返回值。

## 实际应用

**文件操作**：程序读写文件都需要系统调用。

**网络编程**：创建socket、发送接收数据。

**进程管理**：创建进程、等待进程结束。

**权限检查**：检查文件权限、用户身份。

## 总结

- 系统调用是程序请求操作系统服务的接口
- 用户态和内核态区分权限级别
- 系统调用通过中断实现状态切换
- 系统调用有开销，应尽量减少次数
- 库函数通常封装系统调用
