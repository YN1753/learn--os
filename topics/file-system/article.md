# 文件系统

## 概念解释

文件系统是操作系统中**管理数据存储**的部分。它决定了数据如何组织、存储和检索。

你可以把文件系统想象成一个**图书馆管理系统**：
- 文件就像图书馆里的书
- 目录就像书架和分类标签
- 文件系统就是图书管理员，知道每本书放在哪里

## 为什么重要

文件系统让你能够：
- 用人类可读的名字（如"照片.jpg"）访问数据，而不是记住磁盘上的物理位置
- 用目录组织文件，而不是把所有文件堆在一起
- 多个程序安全地共享存储空间
- 在断电后仍然保留数据

## 文件的基本概念

### 什么是文件？

文件是**一组相关数据的集合**，有一个名字和一些属性。

文件属性包括：
- **名称**：人类可读的标识符
- **类型**：文本、图片、程序等
- **大小**：占用的存储空间
- **位置**：在存储设备上的地址
- **保护**：谁可以读取、写入、执行
- **时间**：创建、修改、访问的时间

### 文件操作

- **创建**：分配空间，建立目录条目
- **打开**：将文件信息加载到内存
- **读取**：从文件中获取数据
- **写入**：向文件中保存数据
- **关闭**：释放内存中的文件信息
- **删除**：释放空间，移除目录条目

## 目录结构

### 单级目录

所有文件都在同一个目录中。简单但文件名必须唯一，不适合多用户。

### 两级目录

每个用户有自己的目录。解决了命名冲突，但用户之间不方便共享。

### 树形目录

最常用的方式，像一棵倒过来的树：

```
/
├── home/
│   ├── user1/
│   │   ├── documents/
│   │   └── photos/
│   └── user2/
├── etc/
├── var/
└── tmp/
```

- 根目录（/）是树根
- 每个目录可以包含文件和子目录
- 路径描述文件的位置

### 路径

- **绝对路径**：从根目录开始，如 `/home/user1/documents/report.txt`
- **相对路径**：从当前目录开始，如 `documents/report.txt`

## 磁盘存储

### 磁盘结构

硬盘由多个**盘片**组成，每个盘片有两面，每面有一个**磁头**。

- 盘片旋转 → 磁头移动到指定位置 → 读写数据
- 寻道时间：磁头移动到正确磁道的时间
- 旋转延迟：等待目标扇区转到磁头下的时间

### 磁盘调度算法

多个读写请求时，需要决定执行顺序：

**1. 先来先服务（FCFS）**
- 按请求顺序执行
- 公平但效率低

**2. 最短寻道时间优先（SSTF）**
- 优先执行离当前磁头最近的请求
- 效率高但可能饿死远处的请求

**3. 电梯算法（SCAN）**
- 磁头像电梯一样来回扫描
- 公平且高效

**4. 循环扫描（C-SCAN）**
- 只单向扫描，到头后快速返回
- 更公平的等待时间

## 文件分配方式

### 连续分配

文件占用连续的磁盘块。

优点：读取速度快（顺序读取）。
缺点：文件大小难以扩展，产生外部碎片。

### 链接分配

文件的磁盘块通过指针链接在一起。

优点：没有外部碎片，文件可以动态增长。
缺点：随机访问慢，指针占用空间。

### 索引分配

每个文件有一个索引块，记录所有磁盘块的位置。

优点：支持随机访问，没有外部碎片。
缺点：索引块占用空间，大文件需要多级索引。

## 文件保护

文件系统提供访问控制：

- **所有者**：文件的创建者
- **组**：一组共享权限的用户
- **其他**：所有其他用户

权限类型：
- **读（r）**：查看文件内容
- **写（w）**：修改文件内容
- **执行（x）**：运行程序文件

## 常见文件系统

| 文件系统 | 操作系统 | 特点 |
|----------|----------|------|
| NTFS | Windows | 支持大文件、权限控制、加密 |
| ext4 | Linux | 稳定、高效、支持大容量 |
| APFS | macOS | 优化 SSD、支持快照 |
| FAT32 | 跨平台 | 兼容性好但不支持大文件 |

## 常见错误

**错误1：删除文件就是擦除数据**

删除文件只是移除目录条目，数据仍在磁盘上，直到被新数据覆盖。

**错误2：碎片整理对 SSD 有用**

SSD 没有机械部件，不存在寻道时间问题，碎片整理反而会减少 SSD 寿命。

**错误3：文件系统只管理硬盘**

文件系统也管理 USB 驱动器、SD 卡、甚至内存中的临时文件系统。

## 实际应用

**Windows 资源管理器**：图形化展示文件系统的树形结构。

**回收站**：删除的文件先移到回收站，可以恢复。

**云存储**：Dropbox、Google Drive 使用分布式文件系统。

## 总结

- 文件系统将物理存储抽象为人类可懂的文件和目录
- 树形目录是最常用的组织方式
- 磁盘调度算法优化读写效率
- 文件分配方式影响存储效率和访问速度
- 文件保护确保数据安全
