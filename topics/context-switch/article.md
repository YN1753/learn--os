# 上下文切换

## 概念解释

**上下文切换**是CPU从一个进程切换到另一个进程的过程。操作系统需要保存当前进程的状态（上下文），恢复下一个进程的状态。

就像你看书时被打断：你需要记住看到第几页、做了什么标记，然后才能切换到另一本书。

## 为什么重要

- **并发的基础**：没有上下文切换就没有多任务
- **性能开销大**：每次切换需要几微秒，频繁切换会严重影响性能
- **理解调度的关键**：调度本质上就是决定何时切换

## 上下文包含什么

一个进程的上下文包括：

1. **寄存器状态**：程序计数器(PC)、栈指针(SP)、通用寄存器
2. **内存管理信息**：页表基地址
3. **CPU状态**：中断使能标志、特权级
4. **其他**：浮点寄存器、调试寄存器等

## 切换过程

```
1. 触发切换（时间片用完、IO阻塞、更高优先级进程就绪）
2. 保存当前进程的寄存器到PCB
3. 更新当前进程状态（运行→就绪/阻塞）
4. 调度器选择下一个进程
5. 更新下一个进程状态（就绪→运行）
6. 从PCB恢复下一个进程的寄存器
7. 跳转到下一个进程的PC位置继续执行
```

## 切换触发时机

1. **时间片用完**：当前进程运行时间达到限制
2. **IO阻塞**：进程发起IO请求，需要等待
3. **更高优先级**：有更高优先级的进程就绪
4. **进程结束**：进程正常退出或被杀死

## 可视化说明

在可视化中，你可以：
- 观察上下文切换的完整过程
- 理解PCB（进程控制块）的作用
- 分析切换开销的来源

## 常见错误

1. **切换太频繁**：时间片设置太小，CPU大部分时间在切换
2. **切换太少**：时间片太大，响应性差
3. **忽略缓存影响**：切换后缓存失效，性能下降

## 实际应用

- **Linux**：CFS调度器，时间片根据优先级动态调整
- **实时系统**：尽量减少上下文切换次数
- **虚拟化**：VM Exit/Entry是特殊的上下文切换

## 总结

上下文切换是操作系统实现并发的核心机制。理解切换过程和开销，有助于编写高性能程序和理解调度策略。
