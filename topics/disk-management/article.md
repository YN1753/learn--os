# 磁盘管理

## 概念解释

**磁盘管理**涉及磁盘的分区、格式化和维护。理解磁盘管理有助于合理利用存储空间。

就像管理书柜：先决定分几层（分区），每层放什么类型的书（格式化），然后整理维护。

## 为什么重要

- **存储规划**：合理分配磁盘空间
- **数据安全**：备份和恢复
- **性能优化**：选择合适的文件系统

## 磁盘结构

### 物理结构

- **盘片（Platter）**：存储数据的圆盘
- **磁头（Head）**：读写数据
- **磁道（Track）**：同心圆
- **扇区（Sector）**：磁道的一段，通常512字节
- **柱面（Cylinder）**：所有盘片上相同位置的磁道

### 逻辑结构

磁盘被划分为：
- **主引导记录（MBR）**：第一个扇区
- **分区表**：记录分区信息
- **分区**：逻辑上的独立区域

## 分区方案

### MBR（主引导记录）

- 最多4个主分区
- 或3个主分区+1个扩展分区
- 单分区最大2TB
- 存储在第一个扇区

### GPT（GUID分区表）

- 理论上无限分区
- 支持大于2TB的磁盘
- 冗余备份
- 需要UEFI支持

## 分区操作

### fdisk（MBR）

```bash
fdisk /dev/sda
# 交互式命令
```

### gdisk（GPT）

```bash
gdisk /dev/sda
```

### parted

```bash
parted /dev/sda
```

## 格式化

创建文件系统：

```bash
mkfs.ext4 /dev/sda1      # ext4文件系统
mkfs.xfs /dev/sda1       # XFS文件系统
mkfs.fat -F 32 /dev/sda1 # FAT32文件系统
```

## 挂载使用

```bash
mount /dev/sda1 /mnt/data
# 写入 /etc/fstab 实现开机自动挂载
```

## 磁盘维护

### 检查

```bash
fsck /dev/sda1           # 检查文件系统
```

### 碎片整理

```bash
e4defrag /dev/sda1       # ext4碎片整理
```

### 监控

```bash
df -h                    # 查看磁盘使用
du -sh /path             # 查看目录大小
smartctl -a /dev/sda     # 查看磁盘健康
```

## 可视化说明

在可视化中，你可以：
- 理解磁盘物理结构
- 观察分区过程
- 了解文件系统创建

## 常见错误

1. **分区表损坏**：导致数据丢失
2. **未卸载直接拔出**：可能损坏文件系统
3. **分区大小不当**：后期调整困难

## 实际应用

- **系统安装**：分区和格式化
- **数据备份**：磁盘克隆
- **云存储**：虚拟磁盘管理

## 总结

磁盘管理包括分区、格式化和维护。理解MBR/GPT分区方案和文件系统创建，有助于合理管理存储空间。
