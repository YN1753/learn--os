# 虚拟化技术

## 概念解释

**虚拟化**是在一台物理计算机上创建多个虚拟环境的技术。

想象一下：
- 物理计算机就像一栋大楼
- 虚拟化把大楼隔成多个独立的办公室
- 每个办公室有自己的"虚拟电脑"
- 各个办公室互不干扰

## 为什么重要

虚拟化带来的好处：
- **资源利用率**：一台服务器跑多个应用
- **隔离性**：应用之间互不影响
- **灵活性**：快速创建和销毁环境
- **成本节省**：减少硬件采购
- **便于测试**：随时创建测试环境

## 虚拟化的层次

### 1. 硬件虚拟化

在硬件层面创建虚拟机。

**全虚拟化**：
- 虚拟机完全模拟硬件
- 客户操作系统不需要修改
- 如VMware、VirtualBox

**半虚拟化**：
- 客户操作系统知道自己在虚拟环境中
- 需要修改客户操作系统
- 性能更好

**硬件辅助虚拟化**：
- CPU提供虚拟化支持
- Intel VT-x、AMD-V
- 性能接近原生

### 2. 操作系统虚拟化

在操作系统层面创建隔离的环境。

**容器**：
- 共享宿主机内核
- 比虚拟机更轻量
- 如Docker、LXC

### 3. 应用虚拟化

在应用层面提供虚拟环境。

**语言虚拟机**：
- Java虚拟机（JVM）
- .NET CLR
- 程序在虚拟机上运行

## 虚拟机 vs 容器

| 特性 | 虚拟机 | 容器 |
|------|--------|------|
| 虚拟化层次 | 硬件 | 操作系统 |
| 启动时间 | 分钟级 | 秒级 |
| 资源占用 | 大（每个VM有完整OS） | 小（共享内核） |
| 隔离性 | 强 | 较弱 |
| 性能 | 有损耗 | 接近原生 |
| 适用场景 | 需要不同OS | 应用隔离部署 |

## Docker基础

### 什么是Docker？

Docker是最流行的容器化平台，让应用打包和部署变得简单。

### 核心概念

**镜像（Image）**：
- 应用的只读模板
- 包含代码、运行时、依赖
- 分层存储，可以复用

**容器（Container）**：
- 镜像的运行实例
- 可以启动、停止、删除
- 互相隔离

**仓库（Registry）**：
- 存储和分发镜像
- Docker Hub是最大的公共仓库

### Docker的工作原理

1. **Namespace（命名空间）**：
   - 隔离进程、网络、文件系统
   - 每个容器有自己的命名空间

2. **Cgroups（控制组）**：
   - 限制资源使用（CPU、内存等）
   - 防止容器占用所有资源

3. **UnionFS（联合文件系统）**：
   - 分层存储文件
   - 多个容器可以共享基础层

### Docker的使用

```bash
# 拉取镜像
docker pull nginx

# 运行容器
docker run -d -p 80:80 nginx

# 查看运行的容器
docker ps

# 停止容器
docker stop <container_id>
```

## 虚拟机管理程序（Hypervisor）

### Type 1（裸金属）

直接运行在硬件上，如VMware ESXi、Xen。

**特点**：性能好，适合数据中心。

### Type 2（托管型）

运行在操作系统上，如VirtualBox、VMware Workstation。

**特点**：易于使用，适合个人开发。

## 云计算与虚拟化

虚拟化是云计算的基础：

**IaaS（基础设施即服务）**：
- 提供虚拟机、存储、网络
- 如AWS EC2、阿里云ECS

**PaaS（平台即服务）**：
- 提供运行环境
- 如Heroku、Google App Engine

**SaaS（软件即服务）**：
- 提供软件应用
- 如Gmail、Office 365

## 虚拟化的性能开销

### CPU开销
- 虚拟机需要模拟硬件指令
- 硬件辅助虚拟化减少开销到5%以内

### 内存开销
- 虚拟机需要独立的内存空间
- 内存气球技术可以动态调整

### I/O开销
- 设备模拟有开销
- 直通（passthrough）可以减少开销

## 常见错误

**错误1：容器比虚拟机好**

两者适用场景不同。需要强隔离用虚拟机，需要轻量部署用容器。

**错误2：虚拟化没有性能开销**

虚拟化有开销，但现代技术（硬件辅助虚拟化、容器）已经将开销降到很低。

**错误3：容器就是轻量级虚拟机**

容器共享宿主机内核，隔离性不如虚拟机。安全敏感场景应该用虚拟机。

## 实际应用

**开发环境**：用Docker统一开发环境，避免"我这里能跑"问题。

**测试**：用虚拟机创建干净的测试环境，测试完直接销毁。

**微服务**：每个服务打包成容器，独立部署和扩展。

**灾备**：虚拟机可以快速迁移和恢复。

## 总结

- 虚拟化在一台物理机上创建多个隔离环境
- 虚拟机提供强隔离，容器提供轻量部署
- Docker是最流行的容器化平台
- Namespace、Cgroups、UnionFS是容器的三大技术
- 虚拟化是云计算的基础
- 选择虚拟机还是容器取决于具体需求
