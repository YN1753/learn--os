# 网络协议

## 概念解释

网络协议是计算机网络中数据交换的规则和标准。TCP（传输控制协议）和UDP（用户数据报协议）是传输层最重要的两种协议，分别提供可靠传输和高效传输。

## 为什么重要

- **网络通信基础**：所有网络应用都依赖传输层协议
- **性能与可靠性权衡**：不同场景需要不同协议
- **应用开发必备**：选择合适协议影响应用性能

## 核心原理

### TCP协议特点

| 特性 | 说明 |
|------|------|
| 面向连接 | 三次握手建立连接 |
| 可靠传输 | 确认重传机制 |
| 流量控制 | 滑动窗口机制 |
| 有序传输 | 序列号保证顺序 |
| 拥塞控制 | 慢启动、拥塞避免 |

### UDP协议特点

| 特性 | 说明 |
|------|------|
| 无连接 | 直接发送数据 |
| 不可靠 | 不保证送达 |
| 无序 | 不保证顺序 |
| 高效 | 头部开销小 |
| 支持广播 | 可一对多发送 |

### TCP vs UDP

| 对比项 | TCP | UDP |
|--------|-----|-----|
| 连接 | 面向连接 | 无连接 |
| 可靠性 | 可靠 | 不可靠 |
| 速度 | 较慢 | 较快 |
| 头部大小 | 20字节 | 8字节 |
| 适用场景 | 文件传输、网页 | 视频、游戏、DNS |

## 可视化说明

可视化组件展示了TCP三次握手和四次挥手过程，以及TCP和UDP的数据传输对比。

## 常见错误

1. **盲目选择TCP**：实时应用使用TCP会导致延迟过高
2. **忽视UDP可靠性**：需要在应用层实现可靠性
3. **混淆连接和传输**：连接建立不等于数据可靠

## 实际应用

- **TCP应用**：HTTP/HTTPS、FTP、SMTP、SSH
- **UDP应用**：DNS、DHCP、视频直播、在线游戏、VoIP

## 总结

TCP和UDP是网络通信的基石，理解它们的区别和适用场景对于网络应用开发至关重要。
