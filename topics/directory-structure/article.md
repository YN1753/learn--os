# 目录结构

## 概念解释

**目录**是文件系统的组织结构，它将文件分组管理。目录本身也是文件，包含文件名和inode号的映射。

就像书的目录：通过目录可以快速找到章节的位置。

## 为什么重要

- **组织文件**：将相关文件放在一起
- **快速查找**：通过路径定位文件
- **权限管理**：目录权限控制访问

## 目录结构

### Unix/Linux 目录层次

```
/                   # 根目录
├── bin/            # 基本命令
├── boot/           # 启动文件
├── dev/            # 设备文件
├── etc/            # 配置文件
├── home/           # 用户目录
├── lib/            # 库文件
├── mnt/            # 挂载点
├── proc/           # 进程信息
├── root/           # root用户目录
├── tmp/            # 临时文件
├── usr/            # 用户程序
└── var/            # 可变数据
```

## 路径

### 绝对路径

从根目录开始的完整路径。

```bash
/home/user/documents/file.txt
```

### 相对路径

从当前目录开始的路径。

```bash
./documents/file.txt      # 当前目录下
../other/file.txt         # 上级目录下
```

## 目录操作

### 创建目录

```bash
mkdir directory
mkdir -p path/to/directory    # 递归创建
```

### 删除目录

```bash
rmdir directory               # 删除空目录
rm -r directory               # 递归删除
```

### 列出内容

```bash
ls -la                        # 详细列表
```

### 切换目录

```bash
cd /path/to/directory
cd ..                         # 上级目录
cd ~                          # 主目录
```

## inode

每个文件和目录都有一个inode，存储：
- 文件大小
- 权限
- 所有者
- 时间戳
- 数据块位置

## 挂载

将存储设备连接到目录树：

```bash
mount /dev/sdb1 /mnt/usb
umount /mnt/usb
```

## 可视化说明

在可视化中，你可以：
- 浏览目录树结构
- 理解路径解析过程
- 练习目录操作命令

## 常见错误

1. **路径错误**：绝对路径和相对路径混淆
2. **权限不足**：没有目录的执行权限
3. **循环链接**：符号链接导致无限循环

## 实际应用

- **项目组织**：src、docs、tests目录
- **系统管理**：/etc配置、/var日志
- **容器**：文件系统隔离

## 总结

目录结构是文件系统的骨架。理解目录层次、路径和inode，有助于高效管理文件系统。
