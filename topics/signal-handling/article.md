# 信号处理

## 概念解释

**信号**是操作系统提供的一种进程间通信机制。信号是异步的，可以在任何时候发送给进程，通知它发生了某个事件。

就像手机的推送通知：你可能在做任何事情时收到通知，然后决定如何处理。

## 为什么重要

- **进程控制**：终止、暂停、恢复进程
- **错误处理**：通知进程发生了错误
- **IPC 基础**：简单的进程间通信方式

## 常见信号

| 信号 | 编号 | 默认行为 | 说明 |
|------|------|----------|------|
| SIGHUP | 1 | 终止 | 终端挂起 |
| SIGINT | 2 | 终止 | Ctrl+C |
| SIGQUIT | 3 | 终止+core | Ctrl+\ |
| SIGKILL | 9 | 终止 | 强制终止（不可捕获） |
| SIGSEGV | 11 | 终止+core | 段错误 |
| SIGTERM | 15 | 终止 | 请求终止（可捕获） |
| SIGCHLD | 17 | 忽略 | 子进程状态改变 |
| SIGSTOP | 19 | 暂停 | 暂停进程（不可捕获） |
| SIGCONT | 18 | 继续 | 继续执行暂停的进程 |

## 信号处理方式

### 1. 默认行为

大多数信号的默认行为是终止进程。

### 2. 捕获信号

```c
#include <signal.h>

void handler(int sig) {
    printf("收到信号 %d\n", sig);
}

signal(SIGINT, handler);  // 捕获 Ctrl+C
```

### 3. 忽略信号

```c
signal(SIGINT, SIG_IGN);  // 忽略 Ctrl+C
```

### 4. 恢复默认

```c
signal(SIGINT, SIG_DFL);  // 恢复默认行为
```

## 信号发送方式

### 1. kill() 系统调用

```c
kill(pid, SIGTERM);  // 发送 SIGTERM 给指定进程
```

### 2. 键盘

- Ctrl+C → SIGINT
- Ctrl+\ → SIGQUIT
- Ctrl+Z → SIGTSTP

### 3. 内核

- 除零错误 → SIGFPE
- 段错误 → SIGSEGV
- 非法指令 → SIGILL

## 可视化说明

在可视化中，你可以：
- 观察信号的发送和处理过程
- 理解不同信号的默认行为
- 模拟自定义信号处理函数

## 常见错误

1. **SIGKILL 无法捕获**：不能为 SIGKILL 设置处理函数
2. **信号丢失**：标准信号可能丢失，实时信号不会
3. **竞态条件**：信号处理函数中要避免调用非异步安全函数

## 实际应用

- **优雅终止**：捕获 SIGTERM 进行清理后退出
- **配置重载**：捕获 SIGHUP 重新加载配置文件
- **超时控制**：使用 SIGALRM 实现超时

## 总结

信号是操作系统提供的异步通知机制。理解常见信号和处理方式，有助于编写健壮的系统程序。
