# 性能监控

## 概念解释

**性能监控**是观察和分析系统资源使用情况的过程。通过监控，可以发现性能瓶颈并进行优化。

就像汽车仪表盘：显示速度、油量、温度，让你知道车的状态。

## 为什么重要

- **发现瓶颈**：找到性能问题的根源
- **容量规划**：预测资源需求
- **故障排查**：快速定位问题

## 关键指标

### CPU

- **使用率**：CPU忙碌时间占比
- **负载**：等待运行的进程数
- **上下文切换**：切换频率

### 内存

- **使用率**：已用内存占比
- **缓存命中**：缓存效率
- **交换**：swap使用情况

### 磁盘

- **IO等待**：等待磁盘的时间
- **吞吐量**：每秒读写量
- **IOPS**：每秒IO操作数

### 网络

- **带宽**：网络传输速率
- **延迟**：响应时间
- **丢包率**：丢失的数据包

## 监控工具

### top / htop

实时查看系统状态：

```bash
top        # 基本监控
htop       # 增强版
```

### vmstat

虚拟内存统计：

```bash
vmstat 1   # 每秒刷新
```

### iostat

IO统计：

```bash
iostat -x 1
```

### sar

系统活动报告：

```bash
sar -u 1 10    # CPU使用率
sar -r 1 10    # 内存使用
```

### perf

性能分析工具：

```bash
perf top       # 实时性能分析
perf record    # 记录性能数据
```

## 可视化说明

在可视化中，你可以：
- 观察各种性能指标
- 理解指标的含义
- 学习使用监控工具

## 常见错误

1. **只看平均值**：峰值和分布更重要
2. **忽略IO等待**：可能是主要瓶颈
3. **过度优化**：先找到真正的瓶颈

## 实际应用

- **服务器监控**：Prometheus、Grafana
- **APM**：应用性能监控
- **日志分析**：ELK Stack

## 总结

性能监控是系统管理的基础。通过监控关键指标，可以及时发现和解决性能问题。
