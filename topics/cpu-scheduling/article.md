# CPU调度算法

## 概念解释

CPU调度是操作系统的核心功能之一。当有多个进程需要使用CPU时，调度器决定哪个进程获得CPU时间。

你可以把CPU调度想象成**餐厅服务员**：
- 多桌客人（进程）都在等点菜
- 服务员（调度器）决定先服务哪桌
- 目标是让所有客人都满意（系统效率高）

## 为什么重要

调度算法直接影响：
- **CPU利用率**：CPU有多少时间在真正工作
- **吞吐量**：单位时间完成多少进程
- **响应时间**：用户等待多久才能得到响应
- **公平性**：每个进程是否都能获得CPU时间

## 调度的层次

操作系统有三个层次的调度：

### 1. 长期调度（作业调度）
- 决定哪些进程从外存调入内存
- 控制内存中的进程数量
- 调节系统的并发度

### 2. 中期调度（内存调度）
- 将进程从内存换出到外存
- 当内存紧张时释放空间
- 属于内存管理的一部分

### 3. 短期调度（CPU调度）
- 决定哪个就绪进程获得CPU
- 频率最高（每10-100ms一次）
- 这是通常所说的"调度"

## 调度算法详解

### 1. 先来先服务（FCFS）

**原理**：按进程到达就绪队列的顺序执行。

**特点**：
- 实现简单，用FIFO队列
- 非抢占式
- 对长进程有利

**问题**：护航效应——短进程被长进程阻塞，平均等待时间长。

**适用场景**：批处理系统，对响应时间要求不高。

### 2. 短作业优先（SJF）

**原理**：优先执行预计运行时间最短的进程。

**特点**：
- 平均等待时间最短
- 可能饿死长进程
- 需要预知进程运行时间

**变种**：
- **最短剩余时间优先（SRTF）**：抢占式版本，新进程到来时比较剩余时间
- **高响应比优先**：响应比 = (等待时间 + 服务时间) / 服务时间

**适用场景**：批处理系统，能预知作业时间。

### 3. 时间片轮转（RR）

**原理**：每个进程轮流执行一个时间片。

**特点**：
- 公平，适合交互式系统
- 时间片大小很关键：
  - 太大 → 退化成FCFS
  - 太小 → 上下文切换开销大
- 通常10-100ms

**适用场景**：交互式系统，如桌面操作系统。

### 4. 优先级调度

**原理**：每个进程有优先级，高优先级先执行。

**特点**：
- 可能导致低优先级进程饿死
- 解决方案：老化——逐渐提高等待进程的优先级

**优先级类型**：
- **静态优先级**：创建时确定，不变
- **动态优先级**：根据运行情况调整

**适用场景**：实时系统，有明确优先级需求。

### 5. 多级反馈队列（MLFQ）

**原理**：多个就绪队列，每个队列有不同的优先级和时间片。

**规则**：
1. 新进程进入最高优先级队列
2. 进程用完时间片，降到下一级队列
3. 低优先级队列的时间片更大
4. 高优先级队列优先执行
5. 定期将所有进程提升到最高优先级（防止饿死）

**特点**：
- 兼顾响应时间和吞吐量
- 无需预知进程运行时间
- 最通用的调度算法

**适用场景**：通用操作系统，如Windows、Linux。

### 6. 多级队列调度

**原理**：将进程分类，每类有独立的队列和调度算法。

**例如**：
- 前台进程（交互式）→ 时间片轮转
- 后台进程（批处理）→ FCFS

**特点**：
- 不同类型进程用不同策略
- 队列间可以有固定优先级或时间切分

## 性能指标

| 指标 | 含义 | 计算方式 |
|------|------|----------|
| CPU利用率 | CPU忙碌时间比例 | 忙碌时间/总时间 |
| 吞吐量 | 单位时间完成进程数 | 完成进程数/时间 |
| 周转时间 | 进程从到达到完成的时间 | 完成时间-到达时间 |
| 等待时间 | 进程在就绪队列等待的时间 | 周转时间-执行时间 |
| 响应时间 | 从请求到首次响应的时间 | 首次执行时间-到达时间 |

## 常见错误

**错误1：SJF一定比FCFS好**

SJF平均等待时间最短，但需要预知运行时间，且可能饿死长进程。实际系统往往用更复杂的算法。

**错误2：时间片越小越好**

时间片太小会导致频繁的上下文切换，每次切换都有开销（保存/恢复寄存器等），反而降低性能。

**错误3：优先级调度不需要考虑公平性**

如果不做老化处理，低优先级进程可能永远得不到执行，违反公平性原则。

## 实际应用

**Linux CFS**：完全公平调度器，使用红黑树管理进程，按虚拟运行时间调度。

**Windows**：使用多级反馈队列，32个优先级级别，支持动态优先级调整。

**实时系统**：使用速率单调调度（RM）或最早截止时间优先（EDF）。

## 总结

- CPU调度决定哪个进程获得CPU时间
- 常见算法：FCFS、SJF、RR、优先级、MLFQ
- 每种算法有不同的适用场景和性能特点
- 实际系统通常使用综合算法（如MLFQ）
- 调度算法需要在响应时间、吞吐量和公平性之间平衡
