# 进程组与会话

## 概念解释

**进程组**是一组相关进程的集合，**会话**是进程组的集合。它们用于作业控制和终端管理。

就像一个项目组：组内成员协作，组长负责协调。

## 为什么重要

- **作业控制**：管理前台和后台任务
- **信号传播**：信号可以发送给整个进程组
- **终端管理**：控制终端的进程组

## 进程组

### 概念

- 每个进程属于一个进程组
- 进程组ID（PGID）是组长进程的PID
- 进程组可以有一个控制终端

### 创建

```c
setpgid(0, 0);    // 创建新进程组
```

### 示例

```bash
# 后台运行命令
sleep 100 &
sleep 200 &

# 这两个进程属于同一个进程组
ps -o pid,pgid,cmd
```

## 会话

### 概念

- 会话是一个或多个进程组的集合
- 会话ID（SID）是会话领导者的PID
- 一个会话可以有一个控制终端

### 创建

```c
setsid();    // 创建新会话
```

### 控制终端

- 每个会话可以有一个控制终端
- 前台进程组接收终端输入
- Ctrl+C 发送给前台进程组

## 作业控制

### 前台和后台

```bash
command &        # 后台运行
fg %1            # 切换到前台
bg %1            # 切换到后台
jobs             # 列出作业
```

### 信号处理

```bash
Ctrl+C           # 发送SIGINT给前台进程组
Ctrl+Z           # 发送SIGTSTP给前台进程组
```

## 可视化说明

在可视化中，你可以：
- 理解进程组和会话的关系
- 观察作业控制的过程
- 了解信号传播机制

## 常见错误

1. **孤儿进程组**：父进程退出后，子进程成为孤儿
2. **后台进程输出**：后台进程输出到终端会混乱
3. **僵尸进程组**：进程组中的僵尸进程

## 实际应用

- **Shell**：作业控制
- **守护进程**：脱离控制终端
- **容器**：进程组隔离

## 总结

进程组和会话是进程管理的重要概念。理解它们有助于正确管理后台任务和终端控制。
