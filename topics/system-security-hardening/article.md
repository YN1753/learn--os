# 系统安全加固

## 概念解释

系统安全加固是通过配置和优化操作系统，减少攻击面，提高系统安全性的过程。包括账户安全、文件权限、网络安全、日志审计等方面。

## 为什么重要

- **防御攻击**：减少系统被攻击的风险
- **合规要求**：满足安全合规标准
- **数据保护**：保护敏感数据不被泄露
- **业务连续**：避免安全事件影响业务

## 核心原理

### 安全加固层次

| 层次 | 措施 |
|------|------|
| 账户安全 | 强密码、最小权限、禁用root远程登录 |
| 文件权限 | 设置合理的文件权限、定期审计 |
| 网络安全 | 配置防火墙、关闭不必要端口 |
| 服务安全 | 关闭不必要服务、更新补丁 |
| 日志审计 | 启用日志、定期审查 |

### 常用安全配置

```bash
# 1. 禁用root远程登录
# /etc/ssh/sshd_config
PermitRootLogin no

# 2. 配置防火墙
iptables -A INPUT -p tcp --dport 22 -j ACCEPT
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -j DROP

# 3. 设置密码策略
# /etc/login.defs
PASS_MAX_DAYS 90
PASS_MIN_DAYS 7
PASS_MIN_LEN 12

# 4. 审计日志
auditctl -w /etc/passwd -p wa -k passwd_changes
```

### 安全基线

- **CIS基准**：Center for Internet Security提供的安全配置基准
- **等保标准**：中国信息安全等级保护标准
- **NIST框架**：美国国家标准与技术研究院框架

## 可视化说明

## 常见错误

1. **过度加固**：影响系统正常使用
2. **忽视更新**：不及时安装安全补丁
3. **日志不审计**：有日志但不查看

## 实际应用

- **服务器安全**：云服务器安全配置
- **企业合规**：满足等保要求
- **数据中心**：大规模服务器安全加固
- **个人电脑**：提高个人系统安全性

## 总结

系统安全加固是运维工作的重要组成部分，需要持续关注和改进。
