# 安全模型

## 概念解释

**安全模型**定义了系统如何控制对资源的访问。操作系统通过多种机制保护系统安全。

就像公司的安全系统：门禁控制谁能进入，监控记录谁做了什么。

## 为什么重要

- **保护数据**：防止未授权访问
- **系统完整性**：防止恶意修改
- **合规要求**：满足安全标准

## 访问控制模型

### 1. 自主访问控制（DAC）

资源所有者决定谁能访问。

```bash
chmod 755 file    # Unix权限
```

**优点**：灵活
**缺点**：安全性依赖用户

### 2. 强制访问控制（MAC）

系统强制执行安全策略。

```
安全级别：机密 > 秘密 > 内部 > 公开
规则：高密级不能流向低密级
```

**优点**：更安全
**缺点**：不够灵活

### 3. 基于角色的访问控制（RBAC）

根据角色分配权限。

```
管理员 → 所有权限
普通用户 → 基本权限
访客 → 只读权限
```

**优点**：易于管理
**缺点**：角色设计复杂

## 安全机制

### 1. SELinux

Security-Enhanced Linux，强制访问控制：

```bash
getenforce          # 查看状态
setenforce 1        # 启用
```

### 2. AppArmor

应用程序安全框架：

```bash
aa-status           # 查看状态
```

### 3. 沙箱

隔离进程的执行环境：

```bash
firejail --private firefox    # 沙箱运行
```

### 4. 能力（Capabilities）

细粒度的root权限分割：

```bash
setcap cap_net_raw+ep program
```

## 可视化说明

在可视化中，你可以：
- 理解不同访问控制模型
- 观察安全机制的工作方式
- 了解安全策略的配置

## 常见错误

1. **权限过大**：给不必要的权限
2. **禁用安全机制**：为了方便关闭SELinux
3. **不更新系统**：已知漏洞不修补

## 实际应用

- **服务器**：SELinux/AppArmor保护
- **容器**：namespace隔离
- **云安全**：IAM策略

## 总结

安全模型通过访问控制和安全机制保护系统。理解DAC、MAC、RBAC等模型，有助于设计安全的系统。
