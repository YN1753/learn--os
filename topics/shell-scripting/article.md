# Shell脚本编程

## 概念解释

Shell脚本是将一系列Shell命令写入文件中批量执行的脚本程序。它是Linux/Unix系统管理和自动化的基础工具，可以将重复性的操作自动化执行。

## 为什么重要

- **自动化运维**：批量处理重复性任务
- **系统管理**：管理服务器、配置系统
- **快速开发**：无需编译，快速实现功能
- **管道组合**：灵活组合各种命令

## 核心原理

### Shell类型

| Shell | 路径 | 特点 |
|-------|------|------|
| bash | /bin/bash | 最常用，功能丰富 |
| sh | /bin/sh | POSIX标准，兼容性好 |
| zsh | /bin/zsh | 功能增强，macOS默认 |
| fish | /bin/fish | 用户友好，自动补全 |

### 基本语法

```bash
#!/bin/bash
# 这是注释

# 变量
name="World"
echo "Hello, $name"

# 条件判断
if [ $age -gt 18 ]; then
    echo "成年人"
else
    echo "未成年人"
fi

# 循环
for i in 1 2 3; do
    echo "第 $i 次"
done

# 函数
greet() {
    echo "Hello, $1"
}
greet "Linux"
```

### 常用技巧

1. **管道**：`cmd1 | cmd2` 将cmd1输出作为cmd2输入
2. **重定向**：`>` 覆盖写入，`>>` 追加写入
3. **后台执行**：`cmd &` 后台运行
4. **错误处理**：`set -e` 遇错即停

## 常见错误

1. **忘记shebang**：脚本第一行必须指定解释器
2. **变量未加引号**：`$var` 应该用 `"$var"` 防止分词
3. **空格问题**：`[ $a == $b ]` 等号两边必须有空格

## 实际应用

- **系统管理**：用户管理、备份脚本
- **部署脚本**：自动化部署应用
- **日志分析**：处理和分析日志文件
- **定时任务**：配合cron定时执行

## 总结

Shell脚本是Linux系统管理的必备技能，掌握Shell脚本可以大幅提高工作效率，实现系统管理自动化。
