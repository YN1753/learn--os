# 虚拟内存

## 概念解释

**虚拟内存**是操作系统提供的一种抽象：每个程序认为自己拥有连续的、独立的内存空间，但实际上这些地址可能映射到物理内存的不同位置，甚至在磁盘上。

就像酒店房间号：客人看到的是连续的房间号（虚拟地址），但实际房间可能分散在不同楼层（物理地址）。

## 为什么重要

- **内存隔离**：程序之间互不影响，一个程序崩溃不会影响其他程序
- **内存超分配**：可以运行比物理内存更大的程序
- **简化编程**：程序员不需要关心物理内存布局

## 核心机制

### 1. 分页（Paging）

将虚拟内存和物理内存分成固定大小的块：
- 虚拟内存：页（Page），通常4KB
- 物理内存：帧（Frame），大小与页相同

### 2. 页表（Page Table）

记录虚拟页到物理帧的映射关系。每个进程有自己的页表。

### 3. 地址转换

CPU发出虚拟地址 → 查页表 → 得到物理地址

```
虚拟地址 = [虚拟页号 | 页内偏移]
                ↓ 查页表
物理地址 = [物理帧号 | 页内偏移]
```

### 4. TLB（快表）

页表在内存中，每次访问都要查内存太慢。TLB是页表的高速缓存，加速地址转换。

### 5. 缺页中断

访问的页不在内存中时，触发缺页中断，操作系统从磁盘加载该页。

## 可视化说明

在可视化中，你可以：
- 观察虚拟地址到物理地址的转换过程
- 理解页表的结构和作用
- 模拟缺页中断的处理过程

## 常见错误

1. **混淆虚拟地址和物理地址**：程序使用虚拟地址，硬件使用物理地址
2. **忽略TLB**：TLB miss会导致性能急剧下降
3. **页表太大**：多级页表解决页表过大的问题

## 实际应用

- **Linux**：使用4级页表（PGD→PUD→PMD→PTE）
- **64位系统**：支持48位或57位虚拟地址空间
- **大页（Huge Page）**：减少页表项数量，提高TLB命中率

## 总结

虚拟内存通过分页和页表机制，为每个程序提供独立的地址空间。它是现代操作系统内存管理的基础，使得内存隔离、超分配和简化编程成为可能。
