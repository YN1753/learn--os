# HTTP协议

## 概念解释

HTTP（超文本传输协议）是Web的基础协议，用于客户端和服务器之间的通信。HTTPS是HTTP的安全版本，通过TLS/SSL加密传输数据。

## 为什么重要

- **Web基础**：所有Web应用都基于HTTP协议
- **API通信**：RESTful API使用HTTP方法
- **安全性**：HTTPS保护数据传输安全

## 核心原理

### HTTP请求方法

| 方法 | 用途 | 示例 |
|------|------|------|
| GET | 获取资源 | 获取网页、API数据 |
| POST | 提交数据 | 表单提交、创建资源 |
| PUT | 更新资源 | 更新用户信息 |
| DELETE | 删除资源 | 删除文章 |
| PATCH | 部分更新 | 修改个别字段 |

### HTTP状态码

| 范围 | 含义 | 示例 |
|------|------|------|
| 1xx | 信息 | 100 Continue |
| 2xx | 成功 | 200 OK, 201 Created |
| 3xx | 重定向 | 301永久移动, 302临时移动 |
| 4xx | 客户端错误 | 400错误请求, 404未找到 |
| 5xx | 服务器错误 | 500内部错误, 503服务不可用 |

### HTTP vs HTTPS

| 对比项 | HTTP | HTTPS |
|--------|------|-------|
| 端口 | 80 | 443 |
| 加密 | 明文 | TLS加密 |
| 证书 | 不需要 | 需要SSL证书 |
| 性能 | 较快 | 略慢（加密开销） |
| SEO | 不利 | 有利 |

## 可视化说明

可视化组件展示了HTTP请求-响应过程和常见状态码。

## 常见错误

1. **明文传输敏感数据**：密码等信息必须用HTTPS
2. **忽视缓存控制**：合理使用缓存提高性能
3. **不处理错误状态码**：应正确处理4xx和5xx错误

## 实际应用

- **Web浏览**：浏览器访问网页
- **API开发**：RESTful API设计
- **微服务通信**：服务间HTTP调用
- **CDN加速**：内容分发网络

## 总结

HTTP是互联网的基石，理解HTTP协议对于Web开发和网络编程至关重要。
